# Experimental Environment

## Overview

The experimental environment was designed to ensure reproducibility, isolation, and controlled execution of the proposed architecture.

To achieve this, the infrastructure was fully containerized using Docker and Docker Compose, allowing the coordinated execution of all system components within a single, consistent environment.

This approach minimizes external dependencies and guarantees that the experiments can be replicated under the same conditions.

---

## Infrastructure Composition

The current environment is composed of three main services:

### 1. Hyperledger Besu (Blockchain Node)

Hyperledger Besu is used as the blockchain layer of the prototype.

Main characteristics:

- Ethereum-compatible client  
- running in development mode (`--network=dev`)  
- JSON-RPC interface exposed on port `8545`  
- supports smart contract deployment and execution  

Its role is to provide an immutable and auditable ledger for storing metadata generated by the system.

---

### 2. Eclipse Mosquitto (MQTT Broker)

Mosquitto is responsible for the communication layer.

Main characteristics:

- MQTT protocol (lightweight publish/subscribe)  
- running on port `1883`  
- supports multiple topics and wildcard subscriptions  

Its role is to enable communication between IIoT devices and the orchestrator.

---

### 3. Python Orchestrator

The orchestrator is implemented as a containerized Python service.

Main responsibilities:

- connecting to the Besu node using Web3.py  
- connecting to the MQTT broker using Paho MQTT  
- subscribing to sensor topics (`sensors/#`)  
- processing incoming messages  
- generating hashes and metadata  
- sending transactions to the blockchain  

This component is responsible for integrating the off-chain and on-chain layers.

## Infrastructure Execution

The environment can be started using Docker Compose.

### Start all services

```bash
docker compose up -d
```

Or using the project script:

```bash
./scripts/01_start_infra.sh
```

---

### Verify running containers

```bash
docker compose ps
```

Expected services:

- besu-node  
- mosquitto-broker  
- orchestrator

## Blockchain Validation

The blockchain network was validated using JSON-RPC calls.

### Chain ID verification

```bash
curl -s -X POST http://127.0.0.1:8545 \
-H "Content-Type: application/json" \
--data '{"jsonrpc":"2.0","method":"eth_chainId","params":[],"id":1}'
```

Expected result:

```
0x539
```

This corresponds to chain ID 1337 in decimal format, which is typical for local development networks.

---

### Block progression verification

```bash
curl -s -X POST http://127.0.0.1:8545 \
-H "Content-Type: application/json" \
--data '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'
```

An increasing block number confirms that the blockchain node is active.

---

## MQTT Communication Validation

The MQTT layer can be validated using command-line tools.

### Publish test message

```bash
mosquitto_pub -h localhost -p 1883 \
-t "sensors/sensor_001/data" \
-m '{"sensor_id": "sensor_001", "temp": 23.7, "humidity": 60}'
```

### Subscribe to topic

```bash
mosquitto_sub -h localhost -p 1883 -t "sensors/#" -v
```

Successful communication confirms that the messaging layer is operational.

## Importance of Containerization

The use of Docker provides several advantages for this research:

- standardized execution environment  
- isolation between services  
- simplified deployment  
- reproducibility of experiments  
- easier debugging and monitoring  

These characteristics are essential for academic validation and experimental reliability.

---

## Experimental Relevance

The current environment demonstrates that:

- the infrastructure is stable and reproducible  
- all components can communicate correctly  
- the system supports end-to-end integration  

This establishes a solid foundation for future experiments, including the integration of cryptographic modules such as Attribute-Based Encryption (ABE).
